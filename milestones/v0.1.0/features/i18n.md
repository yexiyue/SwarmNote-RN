# i18n 国际化落地

## 用户故事

作为用户，我希望 App 支持中英文切换，界面语言跟随系统或手动选择。

## 依赖

- 导航 + 页面框架（需要有实际页面才能落地 i18n）
- 主题 + UI 组件库（设置页需要 UI 组件）

## 需求描述

将已配置好的 Lingui i18n 框架在实际页面中落地使用。项目已有 Lingui 配置（`lingui.config.ts`）和空的 locale 目录（`src/locales/zh/`、`src/locales/en/`），需要：

1. 在所有占位页面中使用 `<Trans>` 或 `t` 宏包裹文本
2. 提取消息并完成英文翻译
3. 实现语言切换功能（跟随系统 / 手动选择）
4. 语言偏好持久化

## 交互设计

- 默认跟随系统语言
- 设置页提供语言切换入口（跟随系统 / 中文 / English）
- 切换后即时生效，无需重启

## 技术方案

### 前端

- 使用 `@lingui/react` 的 `<Trans>` 组件和 `useLingui` hook
- 使用 `@lingui/macro` 的 `t` 宏提取消息
- I18nProvider 在根布局中配置
- 语言偏好存储：AsyncStorage

### 工作流

```bash
# 1. 在组件中使用 <Trans> 或 t`` 包裹文本
# 2. 提取消息
pnpm lingui:extract
# 3. 翻译 src/locales/en/ 中的消息
# 4. 编译
pnpm lingui:extract  # Lingui v5 extract 同时编译
```

## 验收标准

- [ ] 所有占位页面的文本均已国际化（无硬编码中文/英文）
- [ ] 中英文切换即时生效
- [ ] 语言偏好可持久化（重启后保持）
- [ ] `pnpm lingui:extract` 无遗漏未提取的消息
