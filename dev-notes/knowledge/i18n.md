# i18n（国际化）

项目使用 **Lingui v6.0.0** + **expo-localization** + **AsyncStorage** 做中英双语切换。源语言 `zh-Hans`，目标 `en`。

## 关键文件

```
src/i18n/
├─ lingui.ts             i18n 实例 + initI18n / setUserLanguage / followSystemLanguage
├─ languageDetector.ts   AsyncStorage + expo-localization 检测链
└─ LinguiProvider.tsx    <I18nProvider defaultComponent={Text}>

src/locales/
├─ zh-Hans/messages.po   源语言（msgid 即 msgstr）
└─ en/messages.po        英文翻译

src/po-types.d.ts        `declare module "*.po"` 类型声明

src/app/settings/
├─ language.tsx          语言选择子页
└─ theme.tsx             主题选择子页（与语言风格对齐）

lingui.config.ts         sourceLocale: zh-Hans, locales: [zh-Hans, en]
babel.config.js          plugins: ['@lingui/babel-plugin-lingui-macro']
metro.config.js          sourceExts += [po, pot] + babelTransformerPath
```

## 工具链链路

```
.tsx 业务代码（含 <Trans>/t``）
   ↓ Metro
   ├─ Babel: @lingui/babel-plugin-lingui-macro 把宏编译成 i18n._() 调用
   └─ Transformer: @lingui/metro-transformer/expo 把 .po 文件转 JS 模块
   ↓
i18n 运行时 (loadAndActivate) → 渲染翻译文本
```

## Lingui v6 升级要点

v6 在 2026-04-22 发布，本项目直接锁 v6。关键改动（已规避）：

- ❌ `@lingui/macro` 包停止维护 → 用 `@lingui/react/macro`（Trans/useLingui）和 `@lingui/core/macro`（msg）
- ❌ `format: "po"` 字符串选项删除 → 不写 `format`，默认即 po
- ⚠️ Node.js 最低 22.19+ → CI 用 `actions/setup-node@v4 + 22`（解析到 22.latest），本地 v24+
- ✅ `@lingui/metro-transformer` 仍是 CJS（其他包都 ESM-only），RN/Expo 兼容性靠这个

**正确做法**：

```ts
import { Trans, useLingui } from "@lingui/react/macro";
import { msg } from "@lingui/core/macro";
```

**不要**：

```ts
import { Trans, t } from "@lingui/macro";  // v6 已停止维护
```

## 常见用法

### JSX 中的文本 → `<Trans>`

```tsx
<Text className="...">
  <Trans>设置</Trans>
</Text>
```

### 属性 / 字符串值 → `t``

```tsx
const { t } = useLingui();
<Pressable accessibilityLabel={t`关闭`}>...</Pressable>
```

### 模块顶层常量 → `msg``

模块顶层不能调 `t``（无 hook context），用 `msg`` 占位 + `i18n._()` 解析：

```tsx
const STATUSES: Record<string, ReturnType<typeof msg>> = {
  online: msg`在线`,
  offline: msg`离线`,
};

function Component() {
  const { i18n } = useLingui();
  return <Text>{i18n._(STATUSES.online)}</Text>;
}
```

### 字符串插值

```tsx
const count = 3;
const text = t`已选 ${count} 个`;       // 自动识别变量并提取 placeholder
```

## 启动顺序

`src/app/_layout.tsx` 必须在第一次渲染前完成 catalog 激活：

```tsx
await Promise.all([
  restoreThemePreference(),
  waitForOnboardingHydration(),
  initAppCore(),
  initI18n(),                  // ← 必须在 ready 之前
]);
```

否则首次渲染会看到 hash msgid 闪烁。

## I18nProvider 的 RN 特殊要求

**必须传 `defaultComponent={Text}`**：

```tsx
<I18nProvider i18n={i18n} defaultComponent={DefaultComponent}>
```

`DefaultComponent` 用 `react-native` 的 `Text` 包裹 children，否则 RN 报错 "Text strings must be rendered within a `<Text>` component"。

## Locale 检测优先级

1. AsyncStorage 中保存的偏好（`language-preference` 键）
2. `expo-localization.getLocales()[0].languageCode`（`zh` 视为 `zh-Hans`）
3. 默认 `zh-Hans`

UI 中"跟随系统"= 不写 AsyncStorage（清掉 key）。

## msgid 策略

**全部用默认 hash（即源中文作为 msgid）**，不显式 `id`。改文案 = 改 key = 旧翻译失效，由 `pnpm lingui:extract` 重新生成。

**不要**：

```tsx
<Trans id="settings.title">设置</Trans>  // 项目暂不立显式 ID 规范
```

## 配置变更后必须 `expo start --clear`

改 `babel.config.js` / `metro.config.js` / `lingui.config.ts` / 安装 `@lingui/*` 后必须 `npx expo start --clear`，否则 Metro 缓存让旧 transformer 残留。

## 提取与编译

```bash
pnpm lingui:extract    # 扫 src/，生成 zh-Hans/en .po；同时编译（Lingui v5+ extract 内置编译）
```

输出统计示例（健康状态）：
```
zh-Hans (source) │     218     │    -
en               │     218     │    0    ← Missing 必须为 0
```

PR 合并前跑一遍，确认 `Missing: 0`。

## 不需要 i18n 的情况

- `editor-test.tsx` 的 sample markdown：是测试**内容**，不是 UI 文本
- 错误信息 `errorMessage(err)`：来自 Rust 后端 / 系统，按原样显示
- 控制台 log / 注释：开发可读性优先
- 编辑器 WebView 内的文本：`packages/editor-web/` 是独立 IIFE bundle，不在 RN macro 链路；编辑器 UI 文本（搜索面板等）暂不需要，等 v0.2.0 #29 实现搜索面板时再决定方案

## 调试

- 编译宏失败 → 检查 `babel.config.js` 是否漏 `@lingui/babel-plugin-lingui-macro`
- `.po` import 失败 → 检查 `metro.config.js` 的 `sourceExts` 和 `babelTransformerPath`
- 运行时显示 hash msgid → catalog 没加载完成就渲染了，检查 `_layout.tsx` 的 `Promise.all`
- TS 报 "*.po 模块找不到" → 检查 `src/po-types.d.ts` 存在
- 切换语言后 UI 不刷新 → 业务代码用了硬编码字符串而非 `<Trans>` / `t``
